ONOS Web API Delelopment

前言

在 ONOS 中，要對 ONOS 系統進行控制，常見的大概有幾種方式，分別是：
+ 擴展 ONOS 的 CLI 命令，透過 ssh 的方式接入 ONOS CLI，並執行相關命令
+ 定義 Network Configuration，透過上傳 JSON 格式的配置文件，來定義 ONOS 的網路配置
+ 透過 ONOS 的 REST API，來對 ONOS 進行控制
今天就要來聊聊，如何對 ONOS 進行 REST API 的開發。

背景知識

在開始之前，先來講一下 RESTful API 的基本概念以及 ONOS 的 REST API 是如何運作的，ONOS 的 REST API 是基於 JAX-RS 的，所以在開發 ONOS 的 REST API 時，需要先了解 JAX-RS 的基本概念。

  • RESTful API 是 http api 的一種設計風格，在 REST 架構中，每一個 URL 代表的是一個資源 (Resource)，資源可能是一個物件、檔案、服務等等，並使用 HTTP method 來代表對資源的操作，如 GET 表示讀取、POST 表示新增、PUT 表示更新、DELETE 表示刪除等等，而在 ONOS 中，一個 WebResource 類別代表了對某一個特定資源的操作 (e.g. devices, flows…)，並實作了對應的一系列 API。

  • Java EE 是 Sun 公司為企業級應用推出的 Java 標準平台，Java EE 定義了一系列 Java API，包括 JDBC、JMS、JNDI、JTA、JSP、Servlet、EJB 等，使開發人員可以基於相同的介面和軟體架構基礎，快速開發與介接。

  • Servlet API 是 Java EE 的其中一個 API 標準，用於伺服器端開發。通常來說我們提到 Servlet 指的是實作了 Servlet 介面的類別，我們可以透過實作 Servlet 介面來完成網路功能的開發。JavaEE 提供了抽象類型 HttpServlet 用於 Web 功能的開發，定義了 doGet、doPost 等 HTTP 方法的函數，又由於 Servlet 被大量用於 Web Server 的開發，因此通常在討論 Servlet 的時候，通常默認在 HTTP 的環境下。

  • 當我們完成了 Servlet 的開發，我們就可以將 Servlet 部署到 Web Server 上，並交，當 webserver 接收並解析 http 請求生成 HttpServletRequest 後，就會根據 URL 來決定要將請求轉發給哪個 Servlet 來處理。

    • 常見的 Servlet container 實作有 Tomcat、Jetty、Glassfish、Wildfly 等。
    • 在 ONOS 內使用的是 GlassFish 作為 HTTP Server。

  • JAX-RS 也是 Java EE 中定義的一個標準，基於 Servlet API，用於創建 RESTful API 的 Web 服務。JAX-RS 主要定意義了一系列的 Annotation，可以將一個 Java class 定義成一個 web resource，提供的 annotation 包跨 Path (定義 resource 的 URL)、GET、POST、PUT、DELETE 等用於標註對應 http method 的處理函數，HeaderParam 等 Param annotion 可以將對應的 parameter 傳遞到指定的處理函數參數上。

  • Jersey 則是一個 JAX-RS 的實作，Jersey 提供了 org.glassfish.jersey.servlet.ServletContainer 這個 Servlet，完成了 WebServer 與 JAX-RS 的整合。當 Jersey 接收到 HTTP request，便會根據 Path annotation 找到對應的 resource class，並將 request 傳遞到對應的處理函數上。
